## Observações relevantes (sem ser falha)

### A11y deep-scan — `/random-nonexistent-xyz` (NotFound page)

Não bloqueia testes (deep-scan é informativo), mas logou **4 violations moderate** que valem refletir:

```
[moderate] landmark-one-main (1 node)
  Document should have one main landmark
  target: html

[moderate] page-has-heading-one (1 node)
  Page should contain a level-one heading
  target: html

[moderate] region (1 node)
  All page content should be contained by landmarks
  target: a (skip-link)

[moderate] skip-link (1 node)
  The skip-link target should exist and be focusable
  target: a href="#main-content"
```

**Análise:** A NotFound page renderiza o skip-link `#main-content`, mas o `<main id="main-content">` não está presente nessa rota. As outras páginas auditadas (`/login`, `/signup`, `/404`, `/offline`) passaram com 0 violations no scan WCAG AA estrito (`06-a11y.spec.ts`). O deep-scan inclui `best-practice` e por isso pega no NotFound real (`/random-nonexistent-xyz`) o que `06-a11y` não pega.

Curioso: `06-a11y.spec.ts` testa `/404` (path literal) com 0 violations, mas `_a11y-deep-scan` em `/random-nonexistent-xyz` (que cai no NotFound via catch-all) tem 4 violations. Possível: `/404` retorna server response específica, enquanto catch-all client-side renderiza componente diferente. Pode ser inconsistência no NotFound.
