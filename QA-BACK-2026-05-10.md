## ⚠️ Auth ausente em endpoints sensíveis

Definição: handler que aceita `clientId` / `workspace_id` no body **e** não importa nenhum de `verifyAuth | tryAuth | authedPost | anonPost | assertClientAccess | assertWorkspaceAccess`.

A maioria absoluta dos handlers (101/179) usa `authedPost` corretamente. Os falsos positivos abaixo SÃO seguros:

- `brand-analysis.ts` / `voice-ingest.ts` / `profile-scraper.ts` / `transcribe-video.ts` / `img-proxy.ts` — adapters SV de 8 linhas que delegam pra handlers reais com auth.
- `images.ts` — adapter SV → roteia pra `image-search` ou `generate-image` (auth no downstream).
- Crons / webhooks / OAuth callbacks (sem auth user é correto).

Casos confirmados sem auth e que merecem revisão:

| Handler                             | Risco                                                                                                                              |
|-------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------|
| `mcp-reader.ts`                     | Usa `MCP_ACCESS_TOKEN` Bearer (não JWT do user). OK para MCP discovery; sem `assertClientAccess` em `tools/call` (que retorna stub). |
| `radar-last-sync.ts`                | GET sem auth. Retorna apenas `MAX(timestamps)` por tabela — sem PII. Aceitável.                                                      |
| `radar-data-{instagram,news,youtube}.ts` | `tryAuth` opcional, exige user mas dados retornados são públicos por nicho (não por cliente). Aceitável.                        |
| `radar-img-proxy.ts`                | Proxy de imagens públicas. Aceitável sem auth.                                                                                       |
| `data-newsletters.ts`               | Exige `tryAuth`, retorna 401 se ausente. OK.                                                                                         |

**Veredicto:** nenhum hole crítico de auth descoberto na auditoria estática. Surface bem coberta.
